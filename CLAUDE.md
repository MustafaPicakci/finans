# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

Kişisel finans paneli: nakit hesapları, düzenli gelir/gider, kredi taksitleri, kredi kartı ekstreleri, nakit akışı takvimi ve canlı fiyatlı çok varlıklı portföy. **Auth: Faz 5.1** (kendi scrypt+session auth'umuz — [auth.ts](apps/server/auth.ts); tüm `/api` guard arkasında, kayıt yalnız ilk owner'a açık). **Çok-kiracılık: Faz 5.2** — tenant tabloları `user_id` ile scope'lu (`prices/price_history` global piyasa verisi; `settings` global fx/tefas vs kullanıcı `user_settings`). Açık kayıt hâlâ kapalı (owner-only); çok-kullanıcı onboarding'i ileride. UI dili Türkçe; kod içi yorumlar ve enum değerleri de Türkçe (`'ALIŞ'`, `'SATIŞ'`).

pnpm workspace monorepo'dur: `apps/server` (Hono API + PostgreSQL), `apps/web` (React/Vite SPA), `packages/engine` (saf finans matematiği, ikisinin de bağımlı olduğu). Bu yapı bilinçli bir evrimin ilk adımı — hedef roadmap için [docs/PLAN.md](docs/PLAN.md)'a bakın (ledger veri modeli, mobil PWA, dosya importu, auth).

## Komutlar

```bash
pnpm install
pnpm dev            # concurrently: API (tsx watch, :8787) + Vite (:5173, /api proxy'li)
pnpm build          # pnpm -r build — her paket kendi build'ini çalıştırır (web: vite build + tsc; server/engine: tsc --noEmit)
pnpm test           # packages/engine vitest testleri (finans matematiği)
pnpm start          # prod: tek süreç, apps/server apps/web/dist statiklerini de sunar (:8787)
```

Tek bir paketi hedeflemek için `pnpm --filter @finans/server dev`, `pnpm --filter @finans/web build`, `pnpm --filter @finans/engine test` gibi filtreler kullanılır. `pnpm build` CI kapısıdır — her paket temiz geçmeli.

Node.js 22+ gerekir. **Veri tabanı PostgreSQL** (Faz 5.0'da `node:sqlite`'tan geçildi) — bağlantı `DATABASE_URL` env değişkeninden (bkz. `apps/server/.env.example`; yerel geliştirme için Docker Postgres önerilir). `apps/web`'de linter yok; `packages/engine`'de vitest var, başka yerde test yok. Eski SQLite verisini taşımak için: `pnpm --filter @finans/server migrate` (bir kez; `data/finans.db` → Postgres, id'ler korunur).

Docker: `docker compose up -d --build`. Compose iki servis çalıştırır: `db` (PostgreSQL, veri `./data/pg` volume'ünde) + `finans` (uygulama, `db`'ye `DATABASE_URL` ile bağlanır, healthcheck ile bekler).

## Mimari

pnpm workspace: backend = `apps/server` (Hono + PostgreSQL), frontend = `apps/web` (React/Vite SPA), paylaşılan finans matematiği = `packages/engine` (saf TS, React'e ve Hono'ya bağımlı değil). `apps/web` bu paketi `@finans/engine` olarak import eder (TS kaynağı doğrudan derlenir, ayrı build adımı gerekmez); `apps/server` şu an için tüketmiyor.

### Para birimi (çok para birimi: TRY + USD)
TRY **taban (reconciliation) para birimidir**; hesaplar/gelir-gider/kart/kredi hep TRY. Yalnızca **portföy işlemleri (`trades`)** bir `currency` (TRY/USD) taşır — kripto/ABD hisse/ETF USD girilebilir. `prices`/`price_history` de sembolün doğal birimini saklar. Pozisyonlar **native** hesaplanır (`positions()` → `Position.currency`; avg/value/K/Z hep o birimde); net varlığa toplanırken `portfolioValueTry(pos, rates)` ile TRY'ye çevrilir (`rates = { usdTry }`, kaynağı `settings.fx_usd_try`). Üstteki **₺/$ görüntü seçici** (App.tsx, localStorage `finans-ccy`) net varlık özeti + KPI'ları nihai TRY rakamından çevirir (`convert(x,"TRY",ccy,rates)`); Nakit Akışı/Rapor/Portföy satırları kendi biriminde kalır (portföy satırları native, projeksiyon TRY). Biçimleyici: `fmtMoney(v, ccy, dec?)` ([theme.ts](apps/web/src/theme.ts)). Yeni para birimi eklemek: `Currency` union + `convert`/`fmtMoney` + FX kuru kaynağı.

### `packages/engine` — saf finans matematiği + tipler
Asıl karmaşıklık burada; React'ten/Hono'dan bağımsız, 46 vitest testiyle korunuyor (`pnpm --filter @finans/engine test`). Yeni bir hesaplama kuralı eklerken önce burada test yaz.
- [types.ts](packages/engine/src/types.ts) — domain tipleri (`AllData`, `Recurring`, `Trade`, ...); `apps/web/src/api.ts` bunları re-export eder.
- [date.ts](packages/engine/src/date.ts) — `hits` (ödeme günü kısa aylarda ay sonuna kayar), `ymOf`/`normYm` (ay biçimi ayrıştırma/doğrulama).
- [projection.ts](packages/engine/src/projection.ts) — `project(data, months)` → günlük `Day[]` nakit projeksiyonu. Hesap bakiyeleri bugünden başlar; recurring/loan/oneoff/kart-ekstresi hareketleri ilgili günlere işlenir. `Day.cashFunds` = o gün elde tutulan **para piyasası fonlarının** TRY değeri (`assets`'in alt kümesi); nakit sayılan fon sembolleri `settings.cash_funds`'tan (virgülle ayrık) okunur. Takvim gün rengini **etkin nakit = `bal + cashFunds`** ile belirler; fonlar Portföy sekmesindeki "nakit say" ile opt-in işaretlenir. Pür-nakit semantiği takvim dışında (Özet en düşük gün, Nakit Liste) korunur.
- [cards.ts](packages/engine/src/cards.ts) — `cardInfos`/`txShares`/`firstCutoff`/`dueOf`: kredi kartı ekstre matematiği. Harcama kesim gününe göre doğru ekstreye düşer, taksitler ardışık ekstrelere bölünür, geçmiş vadeli ekstreler ödenmiş sayılır.
- [loans.ts](packages/engine/src/loans.ts) — `loanRemaining`/`loanActiveOn`: kredinin kalan taksidi tarihten hesaplanır (elle güncelleme yok).
- [portfolio.ts](packages/engine/src/portfolio.ts) — `positions(trades, prices)` → ağırlıklı ortalama maliyetli portföy (her pozisyon kendi `currency`'sinde); pozisyon kapanıp yeniden açılınca maliyet sıfırlanır. `convert(amount, from, to, rates)` (TRY↔USD), `portfolioValueTry(pos, rates)` (net varlık/alokasyon için TRY toplama), `portfolioValueHistory(trades, priceHistory, rates)` (USD-doğal geçmiş fiyatları güncel FX ile TRY'ye çevirir).
- [recurring.ts](packages/engine/src/recurring.ts) — `recActiveOn`: `from_month`/`to_month` aralığına göre aktiflik.

### Backend (`apps/server/`)
- [db.ts](apps/server/db.ts) — **PostgreSQL** (`pg` havuzu, `DATABASE_URL`). Şema `initDb()` içinde `CREATE TABLE IF NOT EXISTS` ile (server başlarken `await initDb()`). `node:sqlite` senkrondu; `pg` async olduğundan `db` yüzeyi de async: `db.all/get/run(sql, ...params)` ve atomik işler için `db.tx(async t => …)`. Eski `?` yer tutucuları db katmanında `$n`'e çevrilir; `INSERT ... RETURNING id` ile `run().id` (lastInsertRowid yerine). `datetime('now','localtime')`/`date('now','localtime')` yerine JS'te üretilen `nowLocal()`/`todayLocal()` parametreleri. `id` kolonları `GENERATED BY DEFAULT AS IDENTITY`, `REAL`→`double precision` (para hassasiyeti korunur). Yeni kolon eklerken `initDb()`'ye `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` deseniyle ekle. Taşıma script'i: [scripts/migrate-sqlite-to-pg.mjs](apps/server/scripts/migrate-sqlite-to-pg.mjs).
- [auth.ts](apps/server/auth.ts) — Faz 5.1 auth: `crypto.scrypt` ile parola hash/verify (sabit-zaman), server-side `sessions` (30 gün, revoke edilebilir). `index.ts`'te `/auth/*` rotaları guard'tan **önce** tanımlanır; `api.use("*")` guard'ı sonraki tüm `/api`'yi korur (401) ve `c.set("user", …)`. Kayıt yalnız `users` boşken (owner bootstrap).
- [index.ts](apps/server/index.ts) — Hono rotaları. Auth guard'ından sonra: tüm okuma **tek** `GET /api/all` ucundan (frontend her mutasyon sonrası hepsini yeniden çeker), kullanıcıya scope'lu (`WHERE user_id=?`; `settings` = global + `user_settings` merge). Yazma tarafı çoğunlukla `crud(route, table, cols)` fabrikasıyla üretilir — **Faz 5.2'de crud tek noktadan tenant-scope'lar** (POST `user_id` enjekte, PUT/DELETE `WHERE id=? AND user_id=?`), yani yeni tablo için CRUD eklemek yine tek `crud(...)` çağrısı ama otomatik izole. Yeni bir tenant tablosu eklerken `TENANT_TABLES`'a da ekle (db.ts — scoping + owner devri onu kullanır). **İstisna: `transactions`** jenerik crud yerine elle yazılmış POST/DELETE kullanır çünkü yan etkisi var — `account_id` verilmişse INSERT hesabın bakiyesini `amount` kadar oynatır (gider −, gelir +), DELETE geri alır; ikisi de `BEGIN/COMMIT` ile atomik, PUT yok (düzenleme = sil + yeniden ekle). Fiyat ve ayar uçları da elle yazılmıştır. Prod'da `serveStatic` ile `apps/web/dist` sunulur (yol `apps/server`'a göre relatif — server her zaman pnpm filter ile kendi dizininden çalıştırılmalı). `node-cron` her 15 dk `refreshAll()` çağırır. **Faz 5.4**: tüm yanıtlara güvenlik başlıkları, `/auth/login|register` in-memory rate-limit (429), KVKK uçları `GET /api/export` (veri indir) + `POST /api/account/delete` (parola onaylı, cascade).
- [prices.ts](apps/server/prices.ts) — Best-effort fiyat çekme. Her varlık türü (`BIST`/`FON`/`ALTIN`/`DOVIZ`/`KRIPTO`/`ETF`) için ayrı fonksiyon; bir kaynak formatını değiştirirse **sadece** ilgili fonksiyon güncellenir. `refreshAll()` yalnızca `trades` tablosunda tutulan sembolleri tazeler, `prices` tablosuna `source='auto'` ile upsert eder. **Para birimi**: fiyat sembolün *doğal* biriminde saklanır — KRIPTO/ETF işlemi USD birimliyse ham USD (`×usdTry` YAPILMAZ), TRY ise (eski) çevrilir; diğer türler TRY. USD/TRY kuru her tazelemede `settings.fx_usd_try`'ye yazılır (portföyde varlık olmasa bile, holding kontrolünden önce — görüntü çevrimi için). Elle girilen fiyatlar `source='manual'`. `ETF` (VOO, QQQ... yurt dışı borsa) Yahoo'dan sorunsuz çekilir. **`FON` (TEFAS)**: resmi API bot-koruması arkasında olduğundan RapidAPI'deki resmi olmayan bir aracı (`RAPIDAPI_KEY` / opsiyonel `RAPIDAPI_KEY_2` env değişkenleri, `.env.example`'a bakın) kullanılır — tek fon sorgusu yok, fon türü başına (1-5) tüm liste çekilip **tamamı** saklanır (kota israf edilmesin diye sadece tutulanlar değil). NAV günde bir hesaplandığından `settings.tefas_last_fetch` ile günde bir kez çağrılır (ücretsiz kota sınırlı); birinci anahtar 429 alırsa otomatik ikinciye geçilir. Anahtar yoksa/kota dolarsa fonlar elle fiyatlanır (Portföy sekmesinde rozet+sıfırla).

### Veri modeli notları
- **Fiyat geçmişi tutulmaz.** `prices` her sembol için tek güncel satır (`PRIMARY KEY (symbol, asset_type)`). Takvimdeki geçmiş günlerin portföy değeri de bugünkü fiyatla değerlenir.
- **Düzenli gelir/gider değişimi** `from_month`/`to_month` (`'YYYY-MM'`, dahil) ile modellenir: tutar değişince eski kayıt bir önceki ayda bitirilir, yeni kayıt o aydan başlatılır — geçmiş projeksiyon bozulmaz.
- **Kredinin kalan taksidi** `first_date` + `total`'dan tarihe göre hesaplanır; biten kredi projeksiyondan kendiliğinden düşer.
- Net varlık = nakit toplamı + Σ(pozisyon değeri TRY'ye çevrili) − toplam kart borcu − kalan kredi borcu (`Σ loan.amount × loanRemaining(loan, bugün)`). Hepsi TRY canonical hesaplanır; görüntü para birimi seçiliyse nihai rakam ona çevrilir.
- **Gerçekleşen işlem (`transactions`) hesaba bağlıysa bakiyeyi oynatır** (sunucu tarafında, atomik) — bir harcama girince ilgili hesabın bakiyesi düşer, silinince geri gelir. Hesap seçilmezse işlem yalnızca Rapor'a girer, bakiyeye dokunmaz. Not: bu Faz 1'in "accounts.balance defterden bağımsızdır" kararının yerini aldı (Faz 4.7). **Hisse/portföy işlemleri (`trades`) hâlâ bakiyeyi otomatik düşürmez** — o akış bilinçli olarak elle bakiye güncellemede kaldı.
- Kayıt düzenleme çoğunlukla yok; sil + yeniden ekle modeli.
